Technical Approach

Leveraging Proven Deep Learning Architectures

WasteAI utilizes state-of-the-art convolutional neural networks (CNNs) to achieve accurate e-waste classification. Our approach leverages two proven architectures that balance performance with practical deployment requirements.

AlexNet

Primary Model
Architecture: 8 learned layers (5 convolutional, 3 fully connected)
Parameters: ~60 million
Key Features: ReLU activation, dropout regularization, local response normalization

Why AlexNet?

  • Proven Track Record: Revolutionized computer vision in 2012, winning ImageNet competition
  • Efficient Training: Simpler architecture means faster training on limited hardware
  • Practical Deployment: Can run on CPU without requiring expensive GPUs
  • Transfer Learning: Pre-trained weights allow fine-tuning for e-waste classification with smaller datasets
  • Student-Friendly: Well-documented with extensive educational resources

ResNet

Comparison Model
Architecture: Deep residual learning with skip connections
Variants: ResNet-18, ResNet-34, ResNet-50
Key Innovation: Residual blocks enable training very deep networks

ResNet as Benchmark

  • Higher Accuracy Potential: Deeper architecture may capture more complex patterns
  • Skip Connections: Helps mitigate vanishing gradient problem in deep networks
  • Performance Comparison: Used to evaluate if added complexity yields meaningful improvements
  • Trade-offs: Requires more computational resources and longer training time

Our Methodology

1

Data Collection & Preparation

Gather diverse e-waste images across multiple categories (phones, laptops, batteries, circuit boards). Augment dataset with rotations, scaling, and lighting variations to improve model robustness.

2

Transfer Learning

Start with pre-trained AlexNet weights from ImageNet. Fine-tune the network on our e-waste dataset, adapting the final classification layers to our specific categories.

3

Training & Optimization

Train models using standard techniques: cross-entropy loss, SGD optimizer, learning rate scheduling. Monitor validation accuracy to prevent overfitting.

4

Comparative Analysis

Benchmark AlexNet against ResNet variants to evaluate accuracy vs. computational efficiency trade-offs for real-world deployment.
